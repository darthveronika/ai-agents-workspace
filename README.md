## Как запустить локально


1. Откройте терминал и клонируйте репозиторий
```
git clone https://github.com/darthveronika/ai-agents-workspace.git
cd ai-agents-workspace
```

2. Установите зависимости
```
npm install
```


3. Запустите dev-сервер
```
npm run dev
```

Приложение откроется на http://localhost:5173. Не требует бэкенда, внешних API или мощного оборудования


## Архитектура решения

Типы (src/types/agent.ts): Строгие интерфейсы (`AgentRole`, `AgentStatus`, `TaskPayload`, `AgentMessage`). Исключают `any`, обеспечивают автодополнение в IDE

Логика (src/hooks/useAgentPipeline.ts): кастомный хук, инкапсулирующий состояние и асинхронный пайплайн. Компоненты только читают данные и вызывают методы

UI (src/App.tsx): декларативная отрисовка, привязка к состоянию хука, обработка формы

Стили (src/index.css): модульные стили на CSS-переменных. Легко кастомизировать, не требуют сборки


## Система агентов

Реализована динамическая маршрутизация задач. Координатор анализирует входной запрос и формирует план выполнения:

- Если задача содержит ключевые слова аналитики (`анализ`, `данные`, `рынок`, `технологии`) → запускается полный пайплайн: `Координатор → Исследователь → Аналитик → Формировщик → Валидатор`
- Если задача общего характера → используется упрощённый пайплайн: `Координатор → Формировщик → Валидатор`

Контекст передаётся между агентами через общий объект `TaskPayload`. Каждый агент модифицирует только свои поля (`collectedData`, `analysis`, `draft`, `finalOutput`), что имитирует изолированную работу в реальных multi-agent системах

1. Координатор: принимает задачу, анализирует тему, формирует план маршрутизации, определяет состав агентов для выполнения
2. Исследователь: получает задачу и тему, собирает релевантные материалы, заполняет поле `collectedData`, обеспечивает фактологическую базу
3. Аналитик: обрабатывает сырые данные из `collectedData`, выделяет тренды и закономерности, формирует поле `analysis`, превращает данные в выводы
4. Формировщик: получает выводы и исходную задачу, структурирует информацию в читаемый формат, заполняет поле `draft`, готовит черновик ответа
5. Валидатор: проверяет черновик на логику и полноту, утверждает финальный результат в поле `finalOutput`, выдаёт пользователю готовый ответ

Взаимодействие построено по принципу Promise-цепочки. Задержки (`setTimeout`) имитируют асинхронные вызовы к внешним сервисам. Архитектура легко адаптируется под `Promise.all` (параллельное выполнение веток) или интеграцию с реальными LLM API
